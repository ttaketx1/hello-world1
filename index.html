Пример работы RSA

  • 1. Простые числа: p = 7, q = 11
  • 2. n = p × q = 7 × 11 = 77
  • 3. φ(n) = (p−1)(q−1) = 6 × 10 = 60
  • 4. Выбираем e: e = 13 (НОД(13, 60) = 1 → ✅)
  • 5. Находим d: такое, что (13 × d) mod 60 = 1 → d = 37

  • Открытый ключ: (n, e) = (77, 13)
  • Закрытый ключ: (n, d) = (77, 37)

  • Сообщение: m = 15 («анализ готов»)
  • Шифрование: C = me mod n = 1513 mod 77 = 71
  • Расшифровка: M = Cd mod n = 7137 mod 77 = 15

⚙️ Объяснение логики работы

  1. Инициализация: загрузка данных, нормализация признаков, разделение на обучающую/тестовую выборки.
  2. Прямой проход: вычисление предсказаний модели на основе текущих весов и входных данных.
  3. Расчёт ошибки: использование функции потерь (например, кросс-энтропия) для оценки отклонения предсказаний от истинных значений.
  4. Обратное распространение: вычисление градиентов функции потерь по всем параметрам модели с помощью цепного правила.
  5. Обновление весов: применение оптимизатора (Adam/SGD) для корректировки параметров в направлении, уменьшающем ошибку.
  6. Валидация: оценка качества на тестовых данных после каждой эпохи для контроля переобучения.
  7. Завершение: остановка при достижении критериев (макс. эпохи, сходимость ошибки, ранняя остановка).

💡 Значения Δ показывают динамику: отрицательное изменение — снижение ошибки (улучшение модели), положительное — рост (требуется настройка гиперпараметров).
